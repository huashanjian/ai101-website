# 项目目标

- 识别校园中可以被智能体改善的真实需求
- 设计一个“感知 → 决策 → 行动”的解决方案
- 通过文档与原型清晰地阐述你的想法

# 交付要求

1. **问题定义**：说明目标用户与核心痛点。
2. **解决方案**：给出智能体的感知方式、决策逻辑与行动流程。
3. **风险与伦理考量**：列出潜在风险，并提出减轻风险的策略。
4. **原型展示**：可使用纸面草图、Figma 或短视频演示。

# 评价维度

- 创意与可行性
- 用户体验与可访问性
- 数据与隐私保护意识
- 项目呈现的完整度
